## 核心需求
- 两个用户（A和B）记录好事坏事，数据实时互通
- 无需登录，双方修改的内容可以互相看到
- 简约清新的视觉风格

## 功能列表
1. 两个按钮：绿色（好事）、红色（坏事）
2. 点击后弹出输入框，输入：事件内容、对象
3. 历史记录列表：时间、内容、记录人、好事/坏事标签、对象
4. 统计面板：用户A好事数、用户A坏事数、用户B好事数、用户B坏事数

## 技术选型
- **前端**: 纯 HTML/CSS/JavaScript（无框架，简单轻量）
- **存储**: Supabase 免费层（支持实时同步）
- **部署**: GitHub Pages

## 设计风格
- 简约清新
- 绿色代表好事，红色代表坏事
- 响应式设计，支持移动端

**Why:** 用户需要一个简单的记录工具，两个用户共享数据，无需复杂功能。
**How to apply:** 保持代码简洁，优先实现核心功能，避免过度设计。
